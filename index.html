The Problem

“

Booking help still felt like filling out a form — even when the person on the other end just wanted to ask a friend.

People need domestic help they can trust — but finding it usually means scrolling long lists and adjusting filters one field at a time: location, availability, price, skill.

The question this redesign answers: what if booking felt like a conversation, instead of a form?

Research & Insights

Behavior

Users don't think in filters ("2BHK, weekday mornings, ₹X"). They think in sentences — the same way they'd ask a friend for a recommendation.

Trust

Verification badges and ratings were the single biggest factor in whether a booking request was completed, not price.

Friction

Manual filtering was rarely abandoned out of confusion — it was abandoned out of fatigue, after 3+ adjustments.

Design Process

Discover

Audit the original

Mapped every existing screen and flagged what already worked before touching anything.

Define

Find the one gap

Manual search was the only real friction point — not the whole app.

Design

Add, don't replace

Layered an AI assistant beside the existing search, never instead of it.

Test

Validate the shortcut

Checked that both paths converge at the same profile and booking screens.

Key Learnings

01

Keep the structure that already works

Revisiting a 6-year-old project forced a useful constraint: don't redesign for its own sake — only add where it genuinely reduces effort.

02

Additive beats disruptive

The AI layer never replaced the manual path — it sat beside it, which made the feature easier to trust and easier to ship.

03

Trust signals matter more than novelty

Verification and rating carried more weight in the original research than any amount of AI polish could.
